f.        Tipo de Elemento:

·         ef_checkbox:  Campo Checkbox (ausencia o presencia de una propiedad)

®          Parámetros:

valor:  valor de la propiedad cuando está seteada (DB);

valor_info:  valor de la propiedad seteada legible para el usuario

·         ef_combo_db:  Combo de opciones generado a partir de un SELECT

®          Parámetros:

sql:  QUERY que carga el combo.  Tiene que devolver 2 columnas (id a grabar en la base y valor legible al usuario);

no_seteado:  Nombre que representa la ausencia de datos (en la base se graba como NULL);

·         ef_combo_db_proyecto: Combo que filtra los elementos según el proyecto ACTUAL.  (Puede incorporar opcionalmente a los del proyecto TOBA)

®     Parámetros:

Sql:  QUERY que carga el combo.  Tiene que devolver 3 columnas (proyecto, id del dato y valor legible al usuario).  Es necesario que se posicione el string ‘%w%’ en el lugar donde se tiene que concatenar el WHERE automático;

columna_proyecto:  columna de la tabla que representa el proyecto;

incluir_toba:  Hay que incluir el proyecto Toba en la lista?

no_seteado:  Nombre que representa la ausencia de dato (en la base se graba como NULL);

·         ef_combo_lista:  Combo basado en una lista explicita.

®     Parámetros:

lista:  La lista representada como un STRING con los elementos separados por COMAS;

no_seteado:  Valor que representa el estado de NO activado;

·         ef_combo_lista_c:  Combo basado en una lista explícita.

®     Parámetros:

lista:  Los pares se separan por “/” y la clave/valor se separa por “,”;

no_seteado:  Valor que representa el estado de NO activado;

·         ef_editable:  Campo de texto editable.

®     Parámetros:

tamano:  Cantidad de caracteres que soporta el campo;

·         ef_editable_clave:  Campo para ingresar contraseñas.

·         ef_editable_fecha:  Campo para ingresar fechas.

·         ef_editable_multilínea:  Campo editable de N líneas.

®     Parámetros:

columnas:  Cantidad de columnas;

filas:  Cantidad de filas;

·         ef_editable_numero:  Campo de texto editable que solo acepta numeros.

·         ef_elemento_ini:  Permite elegir un tipo de elemento e inicializarlo (con una sintaxis similar a la de CSS)

®     Parámetros:

Sql:  El query tiene que devolver 4 campos (Id, Desc. Combo, Descripción y Parámetros);

·         ef_fijo:  Elemento que no cambia de valor.

·         ef_oculto:  Elemento no visible.

·         ef_oculto_proyecto:  Elemento invisible que maneja la clave de proyecto del registro.  Solo permite editar registros del proyecto ACTIVO.

·         ef_oculto_secuencia:  Elemento oculto que maneja una secuencia.

·         ef_oculto_usuario:  Se completa con el usuario que realizó la solicitud.
