Layouts de EIs

Una posible critica a la customizacion del layout es la posible pérdida de uniformidad a las pantallas (algo que se critica a las aplicaciones de escritorio que tienen tantos recursos a su alcanze y tanta personalizacion que terminan siendo interfaces más dificiles que las 'rusticas' web).

Templates Tipo Pull

El template toma más lógica y directamente dirije la creación de la interface. El ejemplo más radical de esta forma es PHP mismo! (no se me ocurre un ejemplo sin que involucre un nuevo mini-lenguaje). Se dice pull porque el template es el activo que saca cosas del sistema: 

<div><?php $this->mostrar_titulo(true, false);?> </div>
<?php foreach ($columnas as $columna) { ?>
<span>....</span>
<?php } ?>

Positivo:

  • Enorme flexibilidad.
  • Bajo overhead.

Negativo:

  • Al carecer de una interface predefinida se deben conocer los interiores de los objetos y por lo tanto no solo es un problema de aprendizaje sino de inestabilidad porque cualquier cambio de implementación rompe la extensión.

Este es el método utilizado hoy en día a travez de herencia y echos en lugar de tags <?php ?>.

Templates Tipo Push

Existe un conjunto de marcas que se van dejando en el HTML que son entendidas por el objeto que lo va a consumir. Por ej un template de un cuadro sería 

<div class='blabla'>
<!--titulo-->
</div>
<div>
<!--grilla-->
</div>
<div>
<!--barra_herramientas-->
</div>

Desde el obtener_html de cada objeto se tomaría este template y se le pondría valor a estas marcas (el template es pasivo). Positivo:

  • Sencillo
  • Definición de una forma más gráfica y menos programática.

Negativo:

  • Generalmente es poco lo que se puede cambiar (depende mucho de la granularidad de las definiciones)
  • El código del generar_html se ensucia un poco y se hace mas lento... hay que parsear el template.

Templates Hibridos

Tomando un poco de cada uno se puede armar uno pasivo pero con parametros dependientes de cada objeto. Por ej un template de un formulario sería: 

<evt id='alta'/>
<div><titulo/></div>
<grilla columnas=4 align='right'/>
<evt id='cancelar'>

Pro:

  • Combinación de los métodos anteriores.

Contra:

  • Además de la contra del overhead ¿Cómo definir el template adecuado? Es difícil encontrar el punto intermedio, cuanto más se deja al template más se flexibiliza pero más se liga a la implementación también.
