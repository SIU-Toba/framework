Inicio / Desarrollo / Refactorizaciones / nucleo La idea basica es separar nucleo, modelo y administrador. Esto deberia requerir una reorganizacion de directorios (Parcialmente en #165)

Nucleo

Runtime enviroment, motor, etc. lo que hace que un item se ejecute.

  • Un item tiene que poder generar distintas salidas, mas alla del HTML.
    • AJAX
      • DAOS (#230)
      • Validaciones en el server
    • Impresion en PDF
    • Exportacion XLS
    • Consola de la operacion (tipo foxpro, para hacer preguntas y demas)
    • Esto implica una refactorizacion grande de la solicitud. (Temas relacionados: zona y tipo de pagina)
  • Constructor de objetos (#117)
    • Cargadores estaticos de informacion de datos del motor
    • Cargadores de array
  • No tienen que haber items del core (ejemplo, respuestas genericas a efs).
  • Deberia ser absolutamente autosuficiente, no necesita al modelo ni al administrador

Modelo

Representacion de todos de todos los elementos que componen al toba

  • Es consumido por el proyecto administrador y por todos los procesos administrativos (utiliza al nucleo para funcionar)
  • Crear clases que representen los elementos fundamentales del modelo del toba (#256)
    • Hay que llegar a definiciones precisas de proyecto, instancia, etc.
  • Repensar los procesos administrativos
    • Exportacion granular a SQL (#58)
    • Precompilacion a arrays PHP
    • Instalacion.
  • Separacion de los metadatos en zonas (#47)
  • Api objeto toba (#129)
  • En la distribucion de directorios, el modelo podria estar un "espacio general" interproyetos. (Un lugar como 'proyectos', limpio a nivel SVN).

Administrador

Proyecto encargado de administrar metadatos

  • Corre sobre el nucleo.
  • trabaja con la ayuda constante del modelo.
  • Funcionamiento erroneo: utilizando el administrador, el "proyecto actual" es "toba" solo si se edita el administrador mismo. Esto esta mal, el proyecto actual es siempre toba (esto afectaba la apertura del ef_popup dentro del admin, porque tomaba el proyecto por defecto). La forma de saber el proyecto que se esta editando dentro del administrador deberia ser otra.
