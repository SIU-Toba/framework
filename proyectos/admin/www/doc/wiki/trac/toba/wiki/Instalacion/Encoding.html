Codificación (Encoding)

Para una introducción corta y sencilla acerca de los encoding y en especial sobre Unicode ver The Absolute Minimum Every Software Developer Absolutely, Positively Must Know About Unicode and Character Sets (No Excuses!)

Actualmente para que los caracteres clásicos utilizados en nuestro lenguaje que superan la barrera del 128 en ASCII (á, ñ, etc.) no se vean como 'cuadraditos', todo el entorno de desarrollo necesita tener un encoding iso88591, esto requiere que:

  • Apache tenga AddDefaultCharset ISO-8859-1 en su configuración (para probar que con Unicode no funciona probar cambiar a AddDefaultCharset UTF-8, reiniciar el servidor y probar alguna página Toba con acentos. Bajo windows es la configuración por defecto.
  • El cliente svn baje los archivos utilizando iso88591. Para distribuciones linux-debian esta configuración se cambia ejecutando como superusuario dpkg-reconfigure locales y seleccionando es_AR.iso88591 o similar (el comando cambia el archivo /etc/enviroment). Bajo Windows no se necesita cambiar nada porque
    • En las versiones 9x no tienen soporte para Unicode
    • En las versiones XP-200x 'adaptan' el texto para que funcionen los programas no-unicode (colaborando también al desentendimiento general del tema).
  • Tanto el cliente como servidor de PostgreSQL necesitan estar configurados para LATIN1 (alias de iso88591). Bajo linux generalmente el servidor toma el encoding del Sistema Operativo, en caso de instalarse linux utilizando Unicode (muy normal actualmente), se necesita explicitar LATIN1 en el comando sql CREATE TABLE, o en el comando de consola createdb (el comando de toba db crear ya esta adaptado a este encoding). También Existe una configuración que afecta a todo el motor en el archivo postgresql.conf.
  • PHP no soporta en forma nativa a Unicode (y al parecer va a ser así hasta la version 6)
