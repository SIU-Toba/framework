Refactorizacion de plantillas css

Sintomas

  1. Los proyectos necesitan definir sus propios estilos y tenerlos bajo su control
  2. Es problable que quieran incluir un css especifico para impresion, en donde se ocultan ciertas cosas, se cambian tipos de letras, etc.
  3. La plantilla css actual pesa mucho (aprox. 60 kb) y esta desordenado (no se sabe que cosas no se usan mas)
  4. Las personalizaciones actuales del css son del tipo todo o nada y haciendo copy-paste
  5. A medida que las versiones de toba pasan se agregan y sacan estilos y los proyectos no tienen chance de seguir estos cambios, si se agrega un estilo nuevo deberian heredarlo inmediatamente.
  6. Hay muchos que se reutilizan en diferentes circunstancias por ejemplo tabla-0 o barra-0-tit, que tendrian mas sentido como por ejemplo obj-filtro y obj-filtro-tit.

Soluciones

  1. Algo esta hecho en [644]
  2. Esto en quilmes esta, se podria tomar la convención de que si esta proyecto_impr se incluye como media=print
  3. A pesar de que el browser la cachea, es muy dificil personalizarlo, esta todo mezclado. Una opción sería dividirlo en partes más pequeñas por ejemplo
    • login.css
    • general.css
    • tipo_pagina.css
    • ei-form.css
    • efs.css
    • ei-cuadro.css
    • ...

Cada pagina podria incluir solamente los que va a usar, el problema es que se deberia saber esto antes de terminar el <head>, y salvo que se use buffering usado ob_start se tendria que pedir temprano los tipos de objetos involucrados (¿seria crear el arbol de objetos no?) antes de que se produzca alguna salida que iria en el body. Si un proyecto quiere personalizar el layout incluiria en su www/css la plantilla personalizada y esta la opción de que se incluya exclusiva o no... (esta ultima podria cargar potencialmente todo dos veces..)

  1. Lo unico bueno de la inclusiva (se incluye el ei-form.css generico y el ei-form.css del proyecto es que los estilos nuevos se incluyen automaticamente en los proyectos) es que permitiria solucionar este tema.
  2. Habria que tomarse el tiempo de redefinir los estilos por ejemplo: 
.obj-form-tit, .obj-cuadro-tit, .obj-filtro-tit {
    //Aca va lo que hay hoy en barra-0-tit
}

Este tipo de agrupacion no seria posible si se definen en archivos separados..
