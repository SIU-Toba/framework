Pasos de la instalación

Para todo lo que es soporte de instalación de Apache, PHP, Postgres y Linux utilizar el  foro del siu.

Guía Rápida Windows

  1. Instalar  Apache.  Instalador recomendado.
  2. Instalar  PHP versión 5.1.2 o superior.  Instalador recomendado (vale aclarar que actualmente se están llevando a cabo algunos cambios menores para lograr compatibilidad total con la versión 5.3.x de PHP):
    • Seleccionar el módulo apache 2.2
    • Indicar el directorio de configuración de Apache (generalmente C:\Archivos de Programa\Apache Software Foundation\Apache2.2\conf
    • No seleccionar todas las extensiones porque no funciona.
    • Seleccionar como mínimo las extensiones PDO PostgreSQL y OpenSSL (esta ultima resuelve un problema de .dlls faltantes de la primera)
    • En caso de que a futuro necesite instalar alguna otra extensión lo puede hacer desde Agregar/Quitar? Programas del Panel de Control de Windows. No borrar el instalador .msi para poder reejecutarlo.
  3. Instalar  PostgreSQL versión 8.3 o menor  Instaladores disponibles.
  4. Instalar la última versión estable de  Subversion para poder utilizar el cliente de linea de comandos. Destildar la opción de instalar los módulos de apache, sólo son necesarios para instalar un servidor svn, no un cliente como este caso. (opcional).
  5. Instalar la última versión estable de  TortoiseSVN para poder utilizar el cliente integrado al explorador de windows. Tener en cuenta que la versión debe ser similar a la del cliente de línea de comandos (opcional).
  6. Instalar la última versión estable de  Graphviz para poder visualizar grafos en el editor (opcional).
  7. Reiniciar el servidor apache para terminar la instalación de php. Navegar hacia  http://localhost verificando que apache funciona.
  8.  Descargar el código fuente de SIU-Toba por ejemplo en el path c:\proyectos\toba
  9. En c:\proyectos\toba\bin, ejecutar instalar.bat y seguir las instrucciones. Al finalizar la instalación no cerrar la ventana sin leer antes las instrucciones de cómo configurar apache. Una vez configurado reiniciar apache.
  10. Para editar el proyecto navegar hacia http://localhost/toba_editor/1.5
  11. Para ejecutar el proyecto navegar hacia http://localhost/miproyecto/1.0 (cambiar la url según el identificador que ha dado al proyecto).
  12. Para poder utilizar la consola nuevamente (para exportar, importar, migrar proyecto, etc.) abrir una consola de windows (cmd.exe) y ejecutar 
    x:\path\a\toba\bin\toba.bat
    Por ejemplo para crear un nuevo proyecto ejecutar 
    x:\path\a\toba\bin\toba.bat proyecto crear -i desarrollo

Alternativa: instalación manual de Apache-PHP

En caso de instalación manual de Apache y PHP (utilizando el ZIP en lugar del instalador)

  1. Agregar al path del sistema (Propiedades del sistema > Opciones Avanzadas > Variables de Entorno > Variables del Sistema > Path)
    • el Path de php, por ej c:\php
    • El path de SVN: c:\Archivos de Programa\Subversion\bin
  2. Verificar la configuración de PHP. En caso de instalación manual copiar el archivo c:\php\php.ini.dist hacia c:\windows\php.ini. Agregar o cambiar las siguientes directivas: (en caso que no se tenga acceso al php.ini se puede crear un archivo .htaccess conteniendo renglones con php_flag clave valor) 
    #Mínimos
magic_quotes_gpc  =  Off
magic_quotes_runtime  =  Off
extension_dir = "c:/php/ext"
extension=php_pdo.dll
extension=php_pdo_pgsql.dll

#Recomendados
error_reporting  =  E_ALL           #Solo para desarrollo
display_errors = On                 #Solo para desarrollo
memory_limit = 128M
post_max_size = 8 M
upload_max_filesize = 8 M
  3. Editar el archivo C:\Archivos de Programa\Apache Group\Apache2\conf\httpd.conf, agregar estas líneas: 
    AddType application/x-httpd-php .php
LoadModule php5_module "c:/php/php5apache2.dll"

Guía para OpenSuse

Distribuciones Debian o derivados

  1. En caso de que los repositorios no tengan una versión actualizada de PHP (5.1.2 o superior) cambiar las fuentes de los paquetes.
  2. Ejecutar como superusuario: 
    apt-get install apache2 libapache2-mod-php5 php5-cli php5-pgsql php5-gd
  3. Ejecutar php -m . En caso de no encontrar los paquetes pdo y pdo_pgsql en el listado, se necesita instalarlos manualmente:
    1. Descargar y compilar las extensiones utilizando el comando pecl: 
      apt-get install php5-dev php-pear postgresql-server-dev-8.1 build-essential
pecl install pdo
pecl install pdo_pgsql
    2. Agregar a los archivos /etc/php5/cli/php.ini y /etc/php5/apache2/php.ini: 
      extension=pdo.so
extension=pdo_pgsql.so
  4. Para instalar servidor y cliente de Postgresql ejecutar (en caso de no contar con un servidor aparte) 
    apt-get install postgresql
  5. Para instalar el cliente Subversion ejecutar 
    apt-get install subversion
  6. Para instalar graphviz ejecutar (opcional) 
    apt-get install graphviz
  7. Editar el archivo /etc/php5/apache2/php.ini y /etc/php5/cli/php.ini: 
    #Mínimos
magic_quotes_gpc  =  Off
magic_quotes_runtime  =  Off

#Recomendados
error_reporting  =  E_ALL      #Solo para desarrollo
display_errors = On                 #Solo para desarrollo
memory_limit = 128M
post_max_size = 8 M
upload_max_filesize = 8 M
  8. Descomprimir o bajar el código fuente de SIU-Toba por ejemplo en el path ~/proyectos/toba
  9. En caso que se haya descargado de un .zip, asignarle permisos de ejecución a los scripts de 'bin': 
    chmod +x ~/proyectos/toba/bin/*
  10. Ejecutar y seguir las instrucciones: 
    ~/proyectos/toba/bin/instalar
  11. Tener en cuenta el esquema de permisos de UNIX. Apache necesita acceso a las carpeta www de Toba y del Proyecto. Si se quieren guardar los logs (opción por defecto) también necesita acceso de escritura a la carpeta instalacion. Existe una comando que acomoda los permisos luego de la instalacion y ejecución: (requiere ejecutarse como usuario privilegiado): 
    sudo ./bin/toba instalacion cambiar_permisos -u <mi_usuario> -g <Usuario Apache generalmente es www-data>
  12. Para editar el proyecto navegar hacia http://localhost/toba_editor/1.5
  13. Para ejecutar el proyecto navegar hacia http://localhost/miproyecto/1.0 (cambiar la url según el identificador que ha dado al proyecto).
  14. Finalmente ver notas sobre la codificación local (encoding)

Notas sobre PostgreSQL

  • Para desarrollo la forma más sencilla de configurar el sistema es dejar el usuario de la base llamado postgres y darle permiso irrestricto desde la misma máquina (una verdero suicidio de seguridad). Para esto editar el archivo /etc/postgresql/8.2/main/pg_hba.conf (linux) o `c:\Archivos de Programa\PostgreSQL\8.2\datapg_hba.conf' y agregar esta línea al final del archivo (cambiar si ya existe con md5) 
    host    all         all         127.0.0.1/32          trust
  • Si se instala versión 8.1 o superior, ver notas sobre la codificación local (encoding).
  • Desde la versión 8.1 postgreSQL  por defecto no permite más joins en forma implícita en los DELETES, para la regeneración y el borrado del proyecto esto es necesario. Para permitirlo setear en true la cláusula add_missing_from de la configuración de postgreSQL.

Alternativa: instalación paso a paso

Una vez descargado o descomprimido el codigo fuente del toba, se puede ejecutar la instalación automática (instalar.bat) o ejecuta la siguiente lista de pasos equivalentes:

  1. Preparar el entorno para la ejecucion de los comandos administrativos.
  2. Configurar la instalacion.
  3. Crear una instancia.
  4. Crear o cargar un Proyecto.
  5. Configurar Apache para publicar el acceso a los proyectos.
  6. Controlar los requisitos del navegador
