Inicio / Instalacion

Pasos de la instalación

Guía Rápida Windows

  1. Instalar la última versión estable de la serie 2.0.x de Apache desde http://httpd.apache.org/download.cgi, con las opciones por defecto.
  2. (OPCIONAL) Instalar la última versión estable de PostgreSQL desde http://www.postgresql.org/download/, con las opciones por defecto.
  3. (OPCIONAL) Instalar la última versión estable de Subversion desde http://subversion.tigris.org/servlets/ProjectDocumentList?folderID=91
  4. Bajar el zip de PHP versión 5.1.2 o superior desde http://ar.php.net/downloads.php
  5. Descomprimir el zip en c:\php
  6. Agregar c:\php en el path del sistema (Propiedades del sistema > Opciones Avanzadas > Variables de Entorno > Variables del Sistema > Path)
  7. Copiar el archivo c:\php\php.ini.dist hacia c:\windows\php.ini y modificar las siguientes directivas:
    • Buscar la línea error_reporting = E_ALL & ~E_NOTICE y reemplazar por error_reporting = E_ALL
    • Buscar la línea magic_quotes_gpc = On y pasarla a magic_quotes_gpc = Off
    • Cambiar extension_dir a extension_dir = "c:/php/ext"
    • Agregar o descomentar estas dos líneas: 
      extension=php_pdo.dll
extension=php_pdo_pgsql.dll
  8. Editar el archivo C:\Archivos de Programa\Apache Group\Apache2\conf\httpd.conf, agregar estas líneas: 
    AddType application/x-httpd-php .php
LoadModule php5_module "c:/php/php5apache2.dll"
  9. Reiniciar el servidor apache para terminar la instalación de php.
  10. Descomprimir o descargar el código fuente de SIU-Toba por ejemplo en el path c:\proyectos\toba
  11. En c:\proyectos\toba\bin, ejecutar instalar.bat y seguir las instrucciones.
  12. Para editar el proyecto navegar hacia http://localhost/toba_editor/
  13. Para ejecutar el proyecto navegar hacia http://localhost/miproyecto (cambiar la url según el identificador que ha dado al proyecto).

Guía Rápida GNU/Linux (distros debian o derivados)

  1. Ejecutar como superusuario: 
    apt-get install apache2 libapache2-mod-php5 php5-cli php5-pdo php5-pdo-pgsql subversion postgresql-8.1 postgresql-client-8.1 graphviz
  2. Editar el archivo /etc/php5/apache2/php.ini y /etc/php5/cli/php.ini:
    • Buscar la línea error_reporting = E_ALL & ~E_NOTICE y reemplazar por error_reporting = E_ALL
    • Buscar la línea magic_quotes_gpc = On y pasarla a magic_quotes_gpc = Off
  3. Descomprimir o bajar el código fuente de SIU-Toba por ejemplo en el path ~/proyectos/toba
  4. Ejecutar y seguir las instrucciones: 
    cd ~/proyectos/toba/bin
./instalar
  5. Para editar el proyecto navegar hacia http://localhost/toba_editor/
  6. Para ejecutar el proyecto navegar hacia http://localhost/miproyecto (cambiar la url según el identificador que ha dado al proyecto).

Detalles de la instalación

Apache

  • URL: http://httpd.apache.org/download.cgi.
  • Versión 2.0.x: Tener en cuenta las precauciones que recomienda PHP.

PHP

  • URL: http://www.php.net/downloads.php.
  • Versión: 5.1.2 o superior
  • Extensiones mínimas necesarias:
    • php_pdo
    • php_pdo_pgsql
  • Configuraciones recomendadas para php.ini distintas a las por defecto:
    • Aumentar el tamaño máximo de memoria a utilizar: memory_limit = 64M
    • No poner requisitos de tiempo máximo de ejecución: max_execution_time = 0
    • Desactivar las 'magic_quotes': magic_quotes_gpc = off. Si no se tiene acceso al php.ini hay una solucion utilizando un archivo .htaccess
    • Aumentar los limites de POST y de tamaños de archivos:
      • post_max_size = 8 M
      • upload_max_filesize = 8 M
    • En desarrollo, mostrar por pantalla todos los sucesos: error_reporting = E_ALL

PostgreSQL

  • URL: http://www.postgresql.org/download/.
  • Versión: 7.4 o mayor.
  • Si se instala versión 8.1 o superior, ver notas sobre la codificación local (encoding).
  • Desde la versión 8.1 postgreSQL por defecto no permite más joins en forma implícita en los DELETES, para la regeneración y el borrado del proyecto esto es necesario. Para permitirlo setear en true la cláusula add_missing_from de la configuración de postgreSQL.

Cliente SVN

El cliente SVN es necesario cuando:

  1. El proyecto se encuentra versionado. Los comandos administrativos que manejan los metadatos van a utilizar el cliente SVN de línea de comandos (svn.exe) para sincronizar los metadatos con el repositorio.
  2. Obtener el código fuente del toba (Para esto es indistinto utilizar un cliente SVN visual o uno de linea de comandos).
  • URL:
    • Cliente de linea de comandos.
    • Cliente recomandado para windows: TortoiseSVN
  • Versión: 1.2 o mayor
  • Si se instala sobre linux, ver notas sobre la codificación local (encoding).

Graphviz (opcional)

Graphviz es una herramienta que permite renderizar esquemas en base a una descripcion de los mismos. La misma constituye la base sobre la que opera el componente ei_esquema, utilizado dentro del toba para mostrar las relaciones entre tablas que constituyen al componente de persistencia datos_relacion. Si bien la utilizacion de la herramienta es opcional, la instalacion esta recomendada porque facilita mucho la utilizacion de este componentes, sobre todo el debug del procesamiento llevado a cabo por el mismo.

  • URL: http://www.graphviz.org/Download.php
  • Versión: 2.5 o superior

Toba

Una vez descargado o descomprimido el codigo fuente del toba, se puede ejecutar la instalación automática o ejecuta la siguiente lista de pasos equivalentes:

  1. Preparar el entorno para la ejecucion de los comandos administrativos.
  2. Configurar la instalacion.
  3. Crear una instancia.
  4. Crear o cargar un Proyecto.
  5. Configurar Apache para publicar el acceso a los proyectos.
