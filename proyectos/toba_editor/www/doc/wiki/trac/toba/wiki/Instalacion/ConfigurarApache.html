Configuración de Apache

Para poder ejecutar tanto el proyecto como el administrador de toba es necesario publicar directorios en el servidor Apache.

Incluyendo a Toba

Toba necesita tener un alias propio, esto permite que los componenentes puedan contar con recursos comunes (principalmente imagenes y javascript) a todos los proyectos de la instalación. Durante la creación de la Instalación se genero un archivo instalacion/toba.conf conteniendo una configuración típica de un Alias de Apache. Este archivo tiene que ser incluído en la configuración general de Apache:

  • Si la instalación de Apache2 se dividió en partes (típico de sistemas linux), se necesita crear un link simbolico en /etc/apache2/sites-enabled: 
    sudo ln -s /home/usuario/proyectos/toba/instalacion/toba.conf  /etc/apache2/sites-enabled/toba
  • Si la instalación es monolitica (típico en sistemas windows) se necesita editar el archivo Apache2\conf\httpd.conf e incluir la línea: 
    ...
include "d:/proyectos/toba/instalacion/toba.conf"
...

El contenido típico de un toba.conf es el siguiente: 

    Alias /toba "/home/usuario/proyectos/toba/www"
    <Directory "/home/usuario/proyectos/toba/www/">
        Options MultiViews
        AllowOverride None
        Order allow,deny
        Allow from all
    </Directory>

El nombre del alias por defecto toba aunque puede ser cualquier otro, en ese caso declararlo con la variable de entorno 'TOBA_ALIAS', por ejemplo: SetEnv TOBA_ALIAS toba_0.9.1.

Incluyendo al editor

	Alias /editor "/home/usuario/proyectos/toba/proyectos/admin/www"
	<Directory "/home/usuario/proyectos/toba/proyectos/admin/www/">
		SetEnv TOBA_DIR "/home/usuario/proyectos/toba"
		SetEnv TOBA_ALIAS toba
		SetENV TOBA_PROYECTO_ALIAS editor
		Options MultiViews
		DirectoryIndex aplicacion.php
		AllowOverride None
		Order allow,deny
		Allow from all
	</Directory>

Incluyendo al proyecto

El proyecto necesita un alias (con el mismo nombre que el proyecto), la manera más sencilla es editar el archivo toba.conf para agregar el alias, por ejemplo para el proyecto mi_proyecto: 

	Alias /mi_proyecto "/home/usuario/proyectos/toba/proyectos/mi_proyecto/www"
	<Directory "/home/usuario/proyectos/toba/proyectos/mi_proyecto/www/">
		SetEnv TOBA_DIR "/home/usuario/proyectos/toba"
		SetEnv TOBA_ALAS toba
		SetEnv TOBA_PROYECTO_ALIAS mi_proyecto
		Options MultiViews
		DirectoryIndex aplicacion.php
		AllowOverride None
		Order allow,deny
		Allow from all
	</Directory>

Si el proyecto consume un toba cuyo alias fue modificado, necesita declarar nuevamente la variable 'TOBA_ALIAS', por ejemplo: SetEnv TOBA_ALIAS toba_0.9.1. Lo mismo sucede si el alias del proyecto difiere de su nombre, se debe incluir SetEnv TOBA_PROYECTO_ALIAS mi_proyecto

Reiniciando el servidor

En ambos casos, luego de cambiar la configuración se debe reiniciar el servidor:

  • En linux: 
    sudo /etc/init.d/apache2 force-reload
  • En windows: 
    net stop apache2
net start apache2
