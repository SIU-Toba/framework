Referencia/ExtensionNucleo/Componentes

Extension de componentes

Aumentar modelo de datos

  • Agragar tablas en DDL
    • Las tablas agregadas tienen que colgar de apex_objeto.
    • Para que el exportador de toba incluya al componente hay que respetar la cabecera "--:"
    • Regenerar la instancia para crear las tablas nuevas.
    • Correr toba nucleo parsear_ddl para actualizar le informacion del modelo utilizada en la exportacion

Registrar componente en el ambiente

  • Crear el editor del componente (clonar algun editor existente)
  • Crear la definicion del tipo del componente.
    • Entrar en la configuracion general del proyecto "toba_editor" y elegir la opcion "Tipos de componente"
    • Agregar el componente y asociar el editor y el datos relacion del mismo (utilizado para la clonacion).

Crear clases PHP

Se necesitan 3 clases:

  • Clase "runtime" (el componente en si).
  • Clase "definicion" (encargada de la carga, exportacion y compilacion de los metadatos del componente).
    • El nombre debe ser igual que la "runtime" con el sufijo "_def".
  • Clase Info (encargada de proveer informacion para el editor).
    • El nombre debe ser igual que la "runtime" con el sufijo "_info".
  • Incluir las 3 clases en el autoload del nucleo.
