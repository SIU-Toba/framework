Objeto árbol

El objeto_ei_arbol tiene como objetivo representar una estructura de árbol en pantalla con posibilidad de incluír funcionalidades de pantallas alrededor de cada nodo del árbol tales como iconos, utilerías, colapsados, ayuda extendida, etc.

Definición

Actualmente la definición del objeto utilizando el administrador es muy básica, todo el comportamiento se define en forma de código.

Carga

El evento carga del ei_arbol espera recibir de parte del CI que lo contiene un objeto que implemente la interface recorrible_como_arbol (Ver detalles de la interface. Este objeto será considerado como el nodo raiz y será el punto de inicio del recorrido a partir de su método hijos().

Eventos predefinidos

  • cambio_apertura: Se dispara cuando al volver al server el objeto descubre que el usuario ha cambiado la apertura de alguno de los nodos. El evento envía un arreglo asociativo 'id_del_nodo' => 0|1 determinando si esta abierto o no.
  • ver_propiedades: Cuando un nodo tiene_propiedades(), el usuario puede clickear sobre el, en este evento se acarrea el identificador del nodo que fue seleccionado.

Ejemplo

Por ahora el catalogo de items, en el frame de la izquierda del administrador, es el ejemplo más claro de este objeto.
