Referencia/Operacion

Operación

La operación se encuentra incluída en un árbol de operaciones dentro de una carpeta particular y su comportamiento está determinado por los componentes contenidos. Estos componentes se definen en forma de árbol con un componente controlador de interface como raiz.

Propiedades básicas

  • Identificador: El identificador permite referirse a la operación desde otro punto del editor o desde el mismo código fuente. Se asigna automáticamente al crearse.
  • Padre: Determina la carpeta donde estará contenida la operación.
  • Tipo de Solicitud: Determina el entorno en donde será ejecutado la operación. El entorno usual es uno Web (Solicitud Web) aunque es posible uno de consola (útil para ejecuciones automáticas o scripts administrativos) o una de acción PHP plana que permite ejecutar código sin relación con el esquema de componentes, menú, javascript, etc.
  • Tipo de página: Permite variar el HTML incluído antes y después del contenido de la operación, por ejemplo para agregar una barra de navegación o un pie de página particular.
  • Menú y Orden: Si la operación se marca como menú se podrá visualizar en la aplicación. Para esto es necesario que las carpetas que lo contienen estén en menú y el usuario logueado tenga permisos sobre estas carpetas y la operación.
  • PHP Plano: La ejecución de la operación delega a la ejecución de un archivo PHP plano, sin poder acceder al esquema de componentes.

Perfiles

Al crear una operación se determina que perfiles tienen acceso a la misma inicialmente.

Navegación entre operaciones

La navegación entre operaciones pasa por un módulo de Toba: el Vinculador. Éste es el encargado de crear la URL, validando que el usuario actual tenga permisos y pasando opcionalmente parámetros hacia la nueva operación.
