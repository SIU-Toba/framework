La idea es incluir carpeta por institución en la carpeta personalizaciones. Luego dentro de cada una de estas carpetas irian los branches con los que se manejan.

  1. La personalizacion se puede crear o bajar. Cuando se crea una personalizacion se toma el branch original de la versión del aplicativo y se hace un COPY hacia un branch de personalizacion. Cada personalización necesita indicar institucion e identificador de personalización. Localmente se hace un checkout de ese branch
  2. En el día a día de la personalización se commitea sobre este branch. La aplicación puede dar un reporte de las cosas modificadas desde la creación.
  3. Cuando se necesita hacer una instalación de una personalización se baja en base a la institución y el identificador.
  4. Cuando existe una nueva versión de la aplicación disponible se realiza:
    1. La personalizacion se encuentra por ejemplo en aplicativo/personalizaciones/unx/1.1.0/trunk
    2. Se hace un copy desde la carpeta versiones/1.2.0 a la aplicativo/personalizaciones/unx/1.2.0/trunk
    3. Se hace un checkout (o switch) a esta carpeta
    4. Se aplica el delta de las dif entre la primer revision y ultima de aplicativo/personalizaciones/unx/1.1.0/trunk
    5. El usuario tiene que revisar y aceptar los cambios utilizando tortoise SVN
