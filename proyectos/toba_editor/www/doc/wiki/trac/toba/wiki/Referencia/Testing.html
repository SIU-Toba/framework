Testing Automático

Dentro de las librerías incluídas en toba se encuentra SimpleTest, esta librería permite armar distintos casos de test que pueden ser ejecutados en forma masiva y automática. Para información general acerca de la utilización de esta librería y sobre la técnica de testing en general se pude encontrar aquí.

Categorías y casos

Un caso de test es un conjunto de pruebas que se hacen generalmente sobre una o más unidades del sistema (generalmente clases). Una categoría es un conjunto de estos casos. Cada proyecto cuenta con una carpeta testing ubicada dentro de la carpeta php. En esta carpeta se pueden definir nuevas categorías agregando carpetas con prefijo test_, por ejemplo test_facturacion. Dentro de estas carpetas se pueden crear casos siguiendo la misma lógica pero esta vez con archivos con este prefijo, por ejemplo test_factura_manual.php.

Para que estos casos sean reconocidos por el entorno, la clases dentro de los casos deben tener el mismo nombre que el archivo y deben heredar de la clase test_toba. Por ejemplo: 

!#php
<?php
class test_factura_manual extends test_toba
{
  ....
}
?>

Ejecución de casos

Hay dos formas de ejecutar los casos de test:

  • A travez del administrador. Desde el frame izquierdo hay un icono para entrar a los casos.
  • Desde consola con el comando proyecto testear y instancia testear
