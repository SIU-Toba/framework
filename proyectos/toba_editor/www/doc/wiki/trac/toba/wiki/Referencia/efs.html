Elementos de Formulario

Un elemento de formulario (abreviado ef) es la representación de un campo de edición. Los distintos tipos de ef en general encapsulan los input clásicos de HTML como un combo, un checkbox o un edit.

Definición

La definición de un ef siempre está manejada por el objeto contenedor (generalmente un formulario). Los atributos básicos que se incluyen son:

  • Identificador: Es la clave con la que se conoce al ef particular en su objeto contenedor.
  • Datos Manejados: Un único ef puede manejar un conjunto de datos, cuando se sabe de antemano la cantidad de datos se puede especificar los identificadores de estos valores separados por coma. En caso de ser uno único se recomiendo usar el mismo dato que el identificador.
  • Colapsado: El elemento y su etiqueta se muestran colapsados por defecto.

Tipos de Elementos

Los efs actualmente se pueden clasificar según la actividad que el usuario realiza:

  • El usuario selecciona:
    • Selecciona un único elemento de un conjunto disponible:
      • checkbox?: Selección entre dos elementos.
      • radios?: Pensado para conjuntos pequeños.
      • combo?: Pensado para conjuntos medianos cuyos elementos son fáciles de encontrar por nombre.
      • popup?: Pensado para conjuntos grandes con métodos de búsqueda complejos
    • Selecciona varios elementos del conjunto:
      • multi_select?: Usando el componente clásico HTML, difícil de entender para usuarios novatos.
      • multi_cruzar?: Cruzando los elementos de un lado al otro.
      • multi_checkbox?: Grupo de checkbox.
      • multi_popup?: Va seleccionando usando un popup.
  • El usuario edita:
    • editable?: Una línea de texto.
    • numero?: Un número.
    • moneda?: Un número representando un importe en una moneda específica.
    • porcentaje?: Un número representando un porcentaje.
    • fecha?: Una fecha, con un calendario de ayuda.
    • editable_mulilinea?: Múltiples líneas libremente, sin formato.
    • editable_formateado?: Edición de texto con formato.
  • El usuario observa un contenido estático
    • fijo?: Muestra su valor como HTML.
  • El usuario sube un archivo:
    • upload: Selecciona un archivo de su sistema para que esté disponible en el servidor.
  • Es invisible al usuario, pero su valor está disponible para el proceso interno:
    • oculto?: Guarda su valor como variable de sesión.
    • oculto_secuencia?: Si no tiene valor, este será determinado por la base de datos.
    • oculto_usuario?: Su valor es el id del usuario actualmente logueado.
    • oculto_proyecto?: Su valor es el id del proyecto actual.

Funcionamiento

El funcionamiento de un ef se puede dividir en tres etapas:

  1. Carga del estado: El objeto que lo contiene le especifica un estado antes de mostrarse en pantalla.
  2. Edición en el cliente: El ef, dependiendo de su tipo, muestra y permite al usuario influir en su valor.
  3. Retorno del estado: En el siguiente pedido, el ef deja disponible al objeto que lo utiliza el nuevo estado, si lo hubiera.

En la edición en el cliente es posible restringir los valores que pueden tomar los efs utilizando las llamadas dependencias entre efs.
Otros comportamientos particulares están a cargo del objeto contenedor, generalmente escuchan los procesamientos de efs.
