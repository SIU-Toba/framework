EF Popup

El ef_popup se utiliza para obtener una combinación clave-valor desde una página popup. Para esto se necesitan dos cosas: La operación o ítem principal y la operación que va a actuar como popup o diálogo.

Popup

El popup es un ítem con el tipo de página popup. Este tipo de pagina mantiene viva la conexión con la ventana original y brinda una función javascript para comunicar los valores seleccionados a la operacion principal: seleccionar(clave, descripcion). El popup puede tener cualquier estructura, a manera de ejemplo se muestra una estructura típica, que consiste de un ci y un cuadro:

El cuadro debe invocar a esta función javascript seleccionar cuando sea seleccionado el elemento. La función javascript seleccionar comunica los nuevos valores y cierra la ventana de popup. Asumiendo que el cuadro maneja una clave que contiene el id y la descripción del registro y que se le definió un evento seleccion a nivel fila, la siguiente extensión javascript posibilita la comunicación con la pantalla original: 

<?php
class cuadro_popup extends objeto_ei_cuadro
{
    function extender_objeto_js()
    {
        echo "
            {$this->objeto_js}.evt__seleccion = function(id) {
                var seleccion = id.split('||');
                seleccionar(seleccion[0], seleccion[1]);
                return false;
            }
        ";
    }
}
?>

Operación principal

En el formulario de la operación se define un ef de tipo ef_popup, y se le indica cual será el item de popup o dialogo que se utilizará:

Al clickear sobre el icono de apertura, se abre el item definido en una ventana popup. Cuando el valor lo carga el usuario el par clave-descripción lo brinda explícitamente desde la selección. Cuando se quiere cargar el ef desde el servidor también se debe explicitar este par, esto se hace definiendo un mecanismo de consulta? (llamado anteriormente DAO). A este mecanismo le llega la clave y debe retornar el valor asociado.
