EF Upload

Un EF Upload presenta un campo de edición (input) para seleccionar cualquier tipo de archivo del sistema de archivos del cliente. En el procesamiento del formulario donde está contenido, el archivo se almacena en el disco del servidor, por lo que se conoce como upload.

Alta

En modo alta el elemento funciona de manera natural, enviando un arreglo con las componentes: name, type, tmp_name, error, size. Si no hubo un error al subir el archivo deberíamos moverlo del directorio temporal a uno propio.

Modificación

En modo modificación, se presenta una diferencia entre el comportamiento esperado para este elemento de interface y el resto. Obviamente no es posible presentar al usuario la ruta del archivo en la máquina cliente, se debe buscar otra opción. Las dos alternativas serían: Esto esta desactualizado, ver la implementación en el trunk

  • Deshabilitar el ef_upload, ejecutando desde el formulario que lo contiene el método establecer_solo_lectura. En este caso hay dos situaciones a distinguir. Si el usuario no seleccionó previamente un archivo, el ef aparecerá deshabilitado. Si tiene un valor, entonces este valor se mostrará en una etiqueta sin dejar la posibilidad de modificación (de acuerdo al comporatmiento de un editable en modo sólo lectura).
  • Permitir la modificación del archivo. Es el comportamiento por defecto, si no establecemos al ef como de sólo lectura. En este caso, cuando el componente tiene un valor se verá en una etiqueta junto a la posibilidad de seleccionar otro, tildando el checkbox Cambiar el Archivo que veremos debajo del nombre del mismo.

La segunda opción intenta emular el modo modificación que para este elemento de formulario en particular se debe tratar de manera especial.

Más información

  • Existe un ejemplo del uso de ef_upload en Referencia, ítem Descripción de Elementos/Formularios/Elementos de formulario/ef_upload.
  • La estructura del estado del ef es la del $_FILES de php.
