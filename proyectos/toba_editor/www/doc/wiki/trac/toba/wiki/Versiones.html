Migración de versiones

Tener en cuenta que los cambios de versión en el 3er dígito (ej. 1.4.x => 1.4.y) no requieren migración, simplemente se actualiza la rama SVN o se pisan los archivos si usan la distribución .zip/.tar.gz.

Preparación

  • Todos los miembros del equipo deben exportar el proyecto y subir cualquier cambio que se quiera mantener 
    toba proyecto exportar -p PROYECTO
  • Opcionalmente crear un nuevo branch del proyecto para poder trabajar sin problemas. Por ejemplo si estamos trabajando en el trunk 
    svn copy http://repositorio.siu.edu.ar/svn/PROYECTO/trunk http://repositorio.siu.edu.ar/svn/PROYECTO/branches/migracion_toba
  • Instalar la nueva versión de toba, respetando el mismo nombre de instancia utilizado en la versión anterior (generalmente es desarrollo)
  • Descargar el branch recientemente creado en la carpeta proyectos de la nueva instalación. Por ejemplo si estamos trabajando en el trunk 
    svn checkout http://repositorio.siu.edu.ar/svn/PROYECTO/branches/migracion_toba proyectos/PROYECTO

Migración del proyecto

  • En la nueva instalación ejecutar el siguiente comando para importar el proyecto: 
    toba proyecto importar -p PROYECTO -d path_al_toba_version_anterior
  • Revisar los cambios realizados automáticamente por el migrador y subirlos al SVN.
  • Opcionalmente es posible importar los metadatos locales (usuarios, fotos del editor, etc.) desde la instancia anterior 
    toba instancia importar -d path_al_toba_version_anterior -r 1
