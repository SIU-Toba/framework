Toba 0.8.3

  • Fecha: 14 de Noviembre de 2005
  • Fuente: source:tags/0.8.3.3
  • Revisiones involucradas: [443] a [975]
  • Plan de la versión
  • Instructivo de migración.
  • Nombre clave: Alarik

A continuación se muestran los cambios más significativos introducidos en esta versión.

Administrador

  • Nuevo editor de items.
  • Nuevos editores para los objetos del 'desarrollo en capas'.
  • Nueva vista lateral del arbol de items. Se muestra un árbol colapsable de items, con capacidad de filtrar por distintos criterios y sacar 'fotos' de este árbol creando vistas personalizadas en forma wysiwyg. Se puede seleccionar un ítem y visualizar su composición en forma integrada. Este componente permite crear una operación en forma top-down.
  • Se integra el testing automático para proyectos Ver más.
  • Nuevo método de clonación de objetos con posibilidad de asignación posterior a otro objeto/item.
  • Capacidad de eliminar un objeto en forma total.

Eventos

  • Los eventos de un objeto se definen a partir del editor del mismo. Para el caso del CI es posible asignarlo a pantallas específicas. La extensión del método get_lista_evento y la utilizacion de los metodos set_eventos() y agregar_evento() dejan de tener motivo salvo cuando se quiere quitar un evento en base a una condición dinámica.
  • Se hace explicita la definición de eventos que se disparan en forma automática.
  • Los eventos pueden clasificarse en grupos y puede definirse el grupo activo en tiempo de ejecucion.

Para más info ver la documentación de Eventos

Objetos

  • Los cuadros y formularios pueden importar su definicion de un objeto_datos_tabla

Nuevos

  • objeto_datos_relacion: Evolución de los db_tablas, permite definir relaciones entre tablas y brinda un mecanismo de sincronizar con distintos medios. Ver documentación
  • objeto_datos_tabla: Evolución de los db_registros y buffers anteriores. Se definen en el administrador a partir de una tabla y es posible asignarles un administrador de persistencia (generalmente uno de BD). Ver documentación
  • objeto_ei_calendario: Calendario para visualizar contenidos diarios y seleccionar días o semanas. Ver documentación.
  • objeto_ei_arbol: Toma una estructura tipo arbol y la grafica con capacidad de colapsar niveles y asociar iconos y utilerías sobre cada elemento. Se utiliza en la nueva vista lateral del administrador. Ver documentación.
  • objeto_ei_archivos: Permite navegar por un sistema de archivos, crear archivos y carpetas y seleccionar archivos con una máscara específica. Se utiliza en el administrador para seleccionar subclases. Ver documentación.

ei_formulario_ml

  • El agregado de una fila puede hacerse en el server.
  • Soporte para permitir o no borrar una fila dada (#193).
  • Soporte para cascadas (excepto creación de fila nueva en el cliente, esto requiere una refactorización del comportamiento de cascadas).
  • Se incluye la totalización como opción para un ef.

Para más info ver la documentación del ei_formulario_ml

ei_cuadro

  • Soporte para definir cortes de control.
  • Soporte para el paginado, a cargo del mismo objeto o a cargo del ci y sus dependencias.
  • Nuevos formateos para mostrar imagenes de toba o del proyecto.
  • Posibilidad de definir funciones de filtrado de eventos por fila (Para los casos en los que la presencia del boton depende de los valores de las fila).
  • El parametro de los eventos del cuadro pasa a ser siempre un array asociativo.

Para más info ver la documentación del ei_cuadro

Efs

  • Nuevo ef_upload, permite subir archivos al server.
  • Nuevo ef_multi_seleccion_check
  • El contenido del ef_popup ahora puede ser editable.
  • El ef_combo_dao puede tener agrupadores, tiene un dao para los valores y otro para las categorias.
  • Los efs se pueden definir colapsados.

Fuentes de datos

  • Se modifica el acceso de conexiones de toba al nuevo esquema. Adodb queda en un segundo plano, aun disponible pero tras una libreria de toba.
  • Las fuentes pueden personalizarse.
  • Actualización de la version de ADOdb (4.6.4).

Varios

  • Cambio de DOCTYPE HTML siguiendo el estandar (#38)
  • La memoria del toba se separa en CELDAS independientes para permitir la ejecucion simultanea de varios CIs en distintos frames.
  • Posibilidad de contar con varias instalaciones de toba corriendo en una misma máquina. Ver Documentación de deployment.
  • Las plantillas css se pueden definir dentro de los proyectos.
  • Se pasa a mantener una sola clase css SIU-Toba, se recomienda mantener las personalizaciones con contenido cercano a esta plantilla (esta previsto a futuro un mecanismo de extensión de plantillas más robusto que el actual).
  • Comando de consola para pasar el sistema a modo mantenimiento.
  • No se necesita más el autoload.
  • El marco, la cabecera, los tipos de paginas y los menus son clases con opción de subclasear por parte de los proyectos. Se soporta la compatibilidad con el esquema anterior
  • Esquema simple de perfil de datos.
