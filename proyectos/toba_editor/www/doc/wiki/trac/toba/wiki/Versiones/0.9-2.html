0.9.1

  • Fecha: 28 de Abril de 2006
  • Fuente: source:tags/0.9.1
  • Revisiones involucradas: [1270] a [1427]
  • Plan de la versión

Cambios más importantes

  • Nucleo: Se incluye un menu css GPL que no trabaja con frames, se configura en las propiedades básicas del proyecto. Se recomienda su uso en lugar de milonic, es mucho mas configurable (a traves de css casi todo) y además es mucho mas performante y no obliga al uso de frames.
  • Núcleo: El uso de frames en la aplicacion es opcional, ya no hay componentes en la arquitectura que lo fuerze.
  • Núcleo: Los logs de codigo se guardan en el sistema de archivos para ser consumidos por el visualizador-analizador de logs.
  • Núcleo: Las cascadas de la solicitud_web se manejan utilizando RPC (también dicho XmlHttpRequest? o simplemente AJAX) en lugar de un frame oculto. La estructura de la comunicación esta en formato JSON. Las cascadas funcionan en ambos esquemas: con frames (esquema viejo) o sin frames (usando RPC) dependiendo del tipo de solicitud.
  • Núcleo: Mejoras al objeto_ei_arbol, ahora el descolapsado de niveles se hace utilizando AJAX.
  • Núcleo: El esquema de encriptado de querystring funciona con los distintos metodos de comunicacion.
  • Admin: Consultas Transversales de items y objetos: Nuevo catalogo de objetos y de items.
  • Admin: Nuevo visualizador-analizador de logs.
  • Núcleo/Admin: Nuevo esquema de vinculacion entre items, asociado al esquema de eventos.

Mejoras menores

  • Logger:
    • No existe mas el logger por pantalla, se va a ser con el de archivos y un analizador
    • Los niveles de log EMEG y ALERT se sacan, para cosas fatales queda CRIT
    • Forma de poder desactivar el logger en un pedido (para que el admin que corre en nombre del proyecto no 'ensucie' el log del mismo)
    • El logger se instancia con un proyecto dado (por defecto el actual)
    • Los mensajes pueden hacer referencia al proyecto del cual surgen
    • Los logs solo se registran si no superan el nivel maximo definido entre el de archivo y el de BD
    • Los logs de archivo un proyecto se guardan en la carpeta logs del mismo (por ej $toba_dir/instalacion/i__desarrollo/p__referencia/logs)
  • Para optimización los servicios pueden enviar cosas previo a la atención de eventos, el obtener_html envia el tipo de pagina superior por ejemplo.
  • Se puede abrir la subclase de una extension usando AJAX.
  • Las clases de impresion hechas por Pablo se pasan a 3eros.
  • El metodo "autovinculo" del vinculador JS pasa a ser "crear_autovinculo" (igual que su par en PHP)
  • Los editores de objetos ante una alta sugieren un nombre del componente en base a su contexto.
  • Comando toba proyecto actualizar (#365)
  • Instancia regenerar pregunta antes si se quiere exportar la info. local
  • Forma de obtener el proyecto actual aun cuando no hay sesion
  • Bug en el cerrado de la sesion, volvia a una URL fija
  • No se podian ejecutar los items de consola
  • Correccion para que el sincronizador de archivos luego de una exportación se utilice solo cuando es necesario.
  • El ci cuando pierde su memoria sincronizada vuelve a la pantalla inicial en forma silenciosa al usuario pero dejando un log.
  • Se sacan los iconos del editor css (usar extensiones de firefox) y el de vinculacion a la ayuda (se hace con la desc. del item) y se agrega el icono para acceder al analizador de logs
  • Nuevo metodo js para abrir poups llamado abrir_popup (reemplaza a popup_abrir_item porque no necesariamente es un item), ademas del comportamiento anterior puede mantener el seguimiento de distintos popups, asi el logger por ejemplo tiene su propio popup que no interfiere con el del ef_popup o la consola js (antes siempre pisaba la misma ventana)
  • El metodo vinculador.autovinculo en js pasa a ser vinculador.crear_autovinculo (igual al vinculador en Php), creo que no entra en la cat. de los otros cambios de api porque no creo que nadie lo haya consumido
  • Se cambia el logo del SIU, la portada inicial del admin, se cambiaron varios iconos y se limpiaron los .psd e imagenes no utilizadas.
  • Se loguean los cambios a objetos e items (#394)
  • La impresion a traves de un iframe funciona en ie y firefox. En opera muestra el dialogo pero imprime la pagina principal. En konqueror no muestra nada y a la segunda vez que se intenta se cuelga :)
  • El objeto_ei_arbol ahora acepta una lista de nodos iniciales (antes asumia un unico nodo inicial), por eso ahora en el carga espera un array de nodos
  • El objeto_ei_arbol carga los nodos colapsados usando ajax, tambien muestra el path actual permitiendo navegarlo. Para poder hacer esto se basa en que los nodos pueden definir si tiene_hijos_cargados, si no los tienen se cargan utilizando ajax y el evt__id__cargar_nodo($id).
  • El armado del path del objeto_ei_arbol no es a traves de un evento sino a traves de la interface recorrible_como_arbol .
  • En la creación automática de subclases ya no se necesita incluir la clase padre.

Notas sobre la versión

El log por pantalla no se utiliza más, en su lugar se puede utilizar el log de archivos y verlo desde el administrador (o directamente desde el sistema de archivos). Este log necesita estar habilitado en el Punto de acceso (admin.php, aplicacion.php): 

define("apex_pa_log_archivo",1);
define("apex_pa_log_archivo_nivel",5);
