Cambios Versión 1.1

  • Fecha: 05/10/07
  • Rama con actualizaciones: source:trunk_versiones/1.1
  • Revisiones trunk involucradas: [2506] a [2955]

Asistentes

Existen una nueva forma de crear una operación completa seleccionando un modelo de operación y completando una configuración mínima. En esta versión se incluyen dos operaciones clásicas que trabajan sobre los datos de una única tabla:

  • ABM Simple: Se crea un filtro,cuadro y formulario tomando como dato de entrada el nombre de la tabla a editar.
  • Grilla: Se crea un formulario_ml que permite editar los contenidos de la tabla completamente en el cliente (ideal para pocos datos o requisitos de alta disponibilidad de edición)

Otros cambios

Nuevas API

  • Utilidades para AJAX. Documentación y ejemplos
  • Nuevo método toba::tabla($nombre, $fuente=null), permite acceder a un datos_tabla (y su extensión) desde cualquier punto de una operación del sistema.
  • El formulario en js tiene dos ventanas antes y despues de las cascadas
  • Ejecución de sentencias (statements) PDO: toba::db()->sentencia($sql, $parametros=null). Shortcut sentencia_fuente($sql, $parametros=null, $id_fuente=null)
  • En Postgresql insert masivo en una tabla a partir de un arreglo: toba::db()->insert_masivo($tabla, $datos). No utiliza PDO, requiere la extensión clásica de postgres.
  • En el ci de js se puden atrapar los eventos generados en los eis (mismo esquema que php). Por ejemplo en la extensión javascript del ci: 
    {$this->objeto_js}}.evt__form__modificacion = function() {
}
  • Nuevo metodo de los eventos para poder obtener su invocacion js pura. Permite poner widgets especificos que invoquen los eventos (no solo botones html). Se puede usar como: 
     $this->pantalla()->evento('mi_evento')->get_invocacion_js();
  • Se pueden quitar operaciones del menú con toba::proyecto()->quitar_item_menu($id_item).
  • Nuevo método get_parametro() del componente, permite obtener los parametros definidos en el editor

Componentes

  • Soporte para campos BLOB en datos_tabla.
  • Soporte para control de valores únicos en el datos_tabla
  • Se pueden definir vínculos en los eventos a nivel de fila de los cuadros. Propagan el id de la fila actual.
  • Implementación de la vista pdf para formularios y formularios_ml (Ver ejemplo en referencia).
  • Nuevos mecanismo de carga para los ef 'seleccionables'. Se puede suministrar un datos_tabla o una consulta_php con el método a invocar.
  • Capacidad de los formularios de mostrar sus tooltips en forma inline
  • Los efs obligatorios pueden relajar su obligatoriedad si son solo-lectura (mismo caso que los ocultos introducido en la 1.0.4)
  • El ml puede ocultar una columna completa.

Editor

  • Los usuarios pueden asociarse a mas de un grupo de acceso.
  • Posibilidad de bloquear usuarios o ips, en base a intentos fallidos de inicio de sesión
  • Nueva vista de 'datos' complementa la vista de 'operaciones' y 'componentes' con listados de fuentes, consultas_php, datos_tabla y datos_relación del proyecto.
  • Eliminación completa de items
  • Mejora en la importación de campos desde un datos_tabla a un ef
  • Un tipo de página del proyecto se puede poner predeterminado, afectando a la creación de la operación y del asistente
  • Se puede consultar las tablas de una fuente de datos desde el mismo editor.

Administración

  • El comando toba puede correr sin variables de entorno seteadas, solo necesita a php en el path
  • Los comandos relacionados con una base de datos están ahora en 'toba base'
  • Nuevo comando toba proyecto publicar y toba proyecto despublicar para crear y borrar el alias del proyecto en el archivo toba.conf
  • Nuevas ventanas de extension del modelo para proyectos. Esto permite que el proyecto se instale junto a la instalación del framework (ver ejemplo en referencia)
  • Los proyectos se pueden mantener en un path relativo a la instalacion
  • Mejoras a la capacidad de merge automático a la exportación SQL de metadatos.
  • Tenemos un nuevo logo!

Migración Manual

  • Cambio de un metodo de la api de los componentes de datos: esta_cargado() se cambia por esta_cargada()
  • vinculador::crear_vinculo se cambia por vinculador::get_url tanto en php como js (se mantiene la compat. hacia atrás). También se hace una pequeña modificación al comportamiento de la zona. Antes no se propagaba el editable entre dos operaciones con igual zona excepto que se explicite, actualmente el editable se pasa excepto entrada por el menú. Si alguna operación confiaba en que la zona se descaraba en base a un vínculo común, ahora debe pasar como 4to parámetros array('zona'=>false) o descargarla manualmente.
  • Al cambiar la cardinalidad de grupos de acceso por usuario, el método toba::usuario()->get_grupo_acceso() se cambia por toba::usuario()->get_grupos_acceso() (plural y retorna un array en lugar de un único valor)
  • Para poder utilizar los comandos 'publicar' y 'despublicar' es necesario cambiar la configuración de instalacion/i__desarrollo/instancia.ini. Por ejemplo: 
    [toba_referencia]
url=/toba_referencia_trunk
  • Una vez migrado, exportado y probado el proyecto, borrar manualmente en las carpetas que no comienzan con 'toba_' en la carpeta 'metadatos/componentes' del proyecto.
