Autodocumentación del código

A medida que las extensiones se van haciendo más importantes, más se hace necesario que los proyectos puedan consumir las interfaces de las cosas que extienden o consumen. Hoy en día casi todo el aprendizaje se hace leyendo directamente las clases o haciendo copy-paste en base a algo que anda.

Documentación en PHP

Un ejemplo de cómo documentar un código esta en source:trunk/php/modelo_documentacion.php

'Buenas?' Prácticas

Una forma rara de programar es empezar con comentarios e ir reemplazandolos por código, al final del día el método/función queda sin comentarios porque todo es claro, por ejemplo esto: 

if ($moneda_hoy != $moneda_ayer && ! $control && nivel($futuro) == niveles::dificil())  { //Estamos en argentina
}

Se convierte en esto: 

$estamos_en_argentina = ($moneda_hoy != $moneda_ayer && ! $control && nivel($futuro) == niveles::dificil());
if ($estamos_en_argentina) { 
}

Y cuando se pone dura la mano el próximo paso es este: 

if (estamos_en_argentina()) { 
}

Críticas

Yo tengo una duda con la autodocumentación, siempre pensaba que estaba fenomenal porque evitaba browsear los fuentes, pero después pasé a ver que viendo los fuentes aprendía bastante y en cierta forma me capacitaba mejor. Desde ya que se necesita un editor que pueda tirar el esqueleto de la clase y pueda hacer un grep rápido por las fuentes y un espíritu aventurero. Muchas veces me ha pasado que recorrer el javadoc era mucho mas mareante que mirar directamente las fuentes (y si no tiene muchos comentarios mejor).

Recien tomé este código en js (hay que imaginar un colorido de javascript): 

/**
 * Representa un evento que será consumido por un CI
 * @param id Identificador del evento, ej: 'modificar'
 * @param validar ¿Se debe validar antes de hacer submit?
 * @param confirmar ¿Se debe confirmar antes de hacer submit?
 * @constructor 
 */
function evento_ei(id, validar, confirmar, parametros) {
	this.id = id;
	this.validar = validar;
	this.confirmar = confirmar;
	this.parametros = parametros;
	this._silencioso = false;
}

Y se convirtió en el HTML que se ve como attachment... a mí me es más fácil ver el código fuente sin todos los /***
