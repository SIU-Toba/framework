Administracion de Instancias

(Esta es una lista de problemas asociados al mantenimiento de Instancias)

¿Como administrar los metadatos de proyectos que se utilizan simultaneamente en varias instancias, desde una unica estacion de trabajo?

  1. (Opcion A) Las informacion se distribuye por proyecto (datos_instancia_{nombre_instancia}.sql)
  2. (Opcion B) Se crea una carpeta "instancias" que posea una carpeta {nombre_instancia} por instancia disponible en la instalación en la que se encuentren un archivo general, y uno por proyecto. Este también es un buen lugar para mover el archivo instancias.php, porque el lugar donde esta hoy ($toba_dir/php) es fácil de pisar sin querer, más cuando a veces nos tenemos que pasar el proyecto por fuera del SVN cuando un checkout es imposible de hacer.

¿Cual es la forma mas efectiva de manejar usuarios globales transversales a muchas instancias?

  • Los datos de las instancias tienen que separarse por zonas. Las zonas deberian poder distribuirse por distintas instancias. De esta forma una instalacion puede apuntar la administracion de usuarios a una instancia, los metadatos a otra, etc.

¿Porqué el archivo de instancias tiene parámetros de conexión a cosas que no son instancias?

  • En ese archivo hay informacion de la "instalacion" que esta muy acoplada a la instancia (Ej: String que se usa para encriptar informacion que esta metida en la instancia), tal vez haya que buscar otro lugar para poner esa info.
