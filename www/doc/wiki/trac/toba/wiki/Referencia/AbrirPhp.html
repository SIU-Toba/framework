Abrir un archivo PHP en un editor externo

Abrir un archivo en el sistema de ventanas del sistema operativo desde un servidor web tiene estas complicaciones:

  • Generalmente apache corre usando un usuario especial y como php corre adentro de este todo proceso disparado desde php se abre utilizando este usuario (generalmente Local Service en windows o www-data en linux), por lo que es probable que no se tomen las configuraciones del usuario de escritorio.
  • Apache corre en background por lo que no es directo hacer que abra un proceso en el sistema de ventanas

Windows

  • Sistemas probados: Windows XP SP2 y Windows 2003 con Apache2
  • Requisitos: Configurar apache para que pueda abrir procesos de escritorio. Para esto ultimo ir a las propiedades del servicio, solapa 'Log On' y activar la opción 'Allow service to interact with desktop'.
  • Particularidades: Como apache corre con un usuario aparte (Local service por defecto) el editor se abre utilizando este usuario, por lo que si el editor soporta el esquema de multiples usuarios tendrá sus configuraciones particulares y abrira una instancia nueva del editor. Por lo general funciona de la siguiente forma:
    • Cuando no hay un editor abierto y se pide abrir el archivo se inicia un editor con el usuario 'Local Service'
    • Cuando hay un editor abierto (con el usuario deseado) y se pide abrir el archivo se reutiliza el editor permaneciendo el usuario.
  • Comando de ejecucion: Se ejecuta un start nombre_archivo y dependiendo de las configuraciones actuales se abre el editor predeterminado.

Linux

  • Sistemas probados: Debian con KDE
  • Requisitos:
    • Primero hay que permitir que un proceso externo pueda abrir algo en el sistema de ventanas, para eso abrir una consola y ejecutar xhost + (esto seguramente deja alguna ventana abierta de seguridad)
    • Una posible solucion al problema de los usuarios es hacer que apache corra con el usuario de escritorio:
        • Cambiar el user y group en httpd.conf
        • agregar HOME=/home/nombre_usuario a la variable ENV del comando apache de /etc/rc.x , esto es para que KDE sepa donde esta el home del usuario.
  • Comando de ejecucion: Se ejecuta export DISPLAY=:0.0;\n kfmclient openURL $archivo, lo primero indica donde se encuentra el sistema de ventana y lo segundo es lo equivalente al start de windows (gnome utiliza gnome-open).
