Consola Administrativa

Para realizar tareas de mantenimiento sobre los distintos proyectos e instancias existe una lista de comandos?.

Configuracion de la consola

  • Agregar el archivo disparador de la consola al PATH (windows: toba.bat, linux: toba)
  • Definir variables de entorno
    • toba_dir (obligatoria) : Directorio donde se encuentra en toba
    • toba_instancia (opcional): Instacia por defecto. Si no existe hay que agregar '-i' en los comandos que referencien instancias.
    • toba_proyecto (opcional): Proyecto por defecto. Si no existe hay que agregar '-p' en los comandos que referencien proyctos.
  • Si los proyectos se versionan bajo SVN, tiene que existir un cliente de lina de comandos accesible por el PATH. Como los componentes de un proyecto se exportan por separado, el comando administrativo encargado de la exportacion tiene que eliminar y agregar archivos de acuerdo a la actividad de la instancia. Si el proyecto se versiona con SVN, el comando mantiene los cambios dialogando directamente con el cliente svn. ( Bajar el SVN, Agregar el SVN al PATH)

Configuración avanzada

Cuando se quiere utilizar el entorno de consola para distintas instalaciones de toba existe una manera más flexible de cumplir con los requisitos. Consiste en no incluir ningún path ni variable relacionada con toba en forma directa sino relegarlo a la ejecución puntual de comandos. Por ejemplo si se cuenta con dos instalaciones totalmente distintas de toba toba_trunk y toba_0.8.2 se harían dos archivos ejecutables: 

#toba_trunk
toba_dir=d:/proyectos/toba
toba_instancia=desarrollo
PATH="/cygdrive/d/proyectos/toba/bin:$PATH"
export toba_dir
export toba_instancia
export PATH

#toba_0.8.2
toba_dir=d:/proyectos/toba_0.8.2
toba_instancia=testing
PATH="/cygdrive/d/proyectos/toba_0.8.2/bin:$PATH"
export toba_dir
export toba_instancia
export PATH

Al iniciar una sesión de sesión es requisito invocar explícitamente alguno de los dos comandos para determinar bajo que instalación se desea ejecutar: 

. toba_trunk

Es importante ejecutar el comando precedido por un punto para que las modificaciones a las variables de entorno se apliquen durante toda la sesión de consola.
