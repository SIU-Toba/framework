Consola Administrativa

Para realizar tareas de mantenimiento sobre los distintos proyectos e instancias existe una lista de comandos? programados en el lenguaje BASH. Estos comandos pueden ejecutarse en forma nativa en sistemas unix o utilizando cygwin bajo windows.

Requisitos

  1. Entorno bash con comandos como awk, egrep, ls y sed.
  2. Cliente php en el path del entorno
  3. Comandos de postgres en el path del entorno (psql, createdb, dropdb)
  4. Carpeta bin de toba en el path del entorno
  5. Variables de entorno para toba:
    • $toba_dir : directorio raiz de la instalación de toba
    • $toba_instancia : nombre de la instancia por defecto que utilizan los distintos comandos

Cumpliendo los requisitos en Cygwin

Cuando se quiere utilizar una sola instalación de toba en una máquina la forma fácil de cumplir los requisitos es la siguiente:

  1. Instalar el cygwin con las opciones por defecto
  2. Agregar el path de php a windows, cygwin lo hereda
  3. Instalar una version de postgresql para windows y agregar la carpeta bin al path de windows
  4. Agregar el directorio bin de toba al path de windows
  5. Agregar las variables al archivo etc/profile dentro del arbol de instalación de cygwin: 
    toba_dir=d:/proyectos/toba
export toba_dir
toba_instancia=desarrollo
export toba_instancia

Configuración avanzada

Cuando se quiere utilizar el entorno de consola para distintas instalaciones de toba existe una manera más flexible de cumplir con los requisitos. Consiste en no incluir ningún path ni variable relacionada con toba en forma directa sino relegarlo a la ejecución puntual de comandos. Por ejemplo si se cuenta con dos instalaciones totalmente distintas de toba toba_trunk y toba_0.8.2 se harían dos archivos ejecutables: 

#toba_trunk
toba_dir=d:/proyectos/toba
toba_instancia=desarrollo
PATH="/cygdrive/d/proyectos/toba/bin:$PATH"
export toba_dir
export toba_instancia
export PATH

#toba_0.8.2
toba_dir=d:/proyectos/toba_0.8.2
toba_instancia=testing
PATH="/cygdrive/d/proyectos/toba_0.8.2/bin:$PATH"
export toba_dir
export toba_instancia
export PATH

Al iniciar una sesión de sesión es requisito invocar explícitamente alguno de los dos comandos para determinar bajo que instalación se desea ejecutar: 

. toba_trunk

Es importante ejecutar el comando precedido por un punto para que las modificaciones a las variables de entorno se apliquen durante toda la sesión de consola.
