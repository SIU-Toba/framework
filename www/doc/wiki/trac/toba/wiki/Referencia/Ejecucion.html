Seguimiento de la ejecución

A continuación se van a listar los aspectos más importantes de le ejecución toba en un entorno Web.

Conceptos Previos

Para desarrolladores nuevos a las aplicaciones Web surge un nuevo concepto llamado request o pedido que marca la comunicación entre el lado cliente y servidor.
Al no permanecer en memoria entre pedidos, PHP almacena un entorno mínimo de datos en las llamadas variables de sesión, por ejemplo mantienen el usuario logueado y toda aquella información persistente durante la sesión del usuario. Toda otra información es descartada cuando la página termina de servirse, siendo necesario re-ejecutar en cada pedido (esto implica parsear e interpretar nuevamente) para volver a disponer de estos datos.

Ingreso

Las sesiones de trabajo en Toba siempre pasan a través de un punto de acceso, esto se conoce en la industria como FrontController dejando de lado el PageController clásico en aplicaciones PHP. La URL que ingresa el usuario direcciona este punto de acceso, por ejemplo: 

http://localhost/mi_proyecto/aplicacion.php

Es por este archivo (aplicacion.php) por donde pasarán los siguientes pedidos, recibiendo como parámetros la operación y atributos propios de esa operación entre otras cosas. En este punto de acceso están las primeras configuraciones de ejecución, incluyen la instancia a ejecutar.
Cuando las variables de sesión no están presentes es signo de que el usuario no está logueado y se produce la autentificación.

Vinculación a la Instancia

La instancia determina el conjunto de usuarios que serán utilizados en esta sesión.
