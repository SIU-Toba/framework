Fuente de Datos

Una fuente de datos es una entidad que referencia a un motor de base de datos relacional y permite operar sobre este independientemente del vendedor específico al que pertenece. Esto no quiere decir que el lenguaje con el que se accede se abstrae o compatibiliza, el lenguaje sigue siendo SQL y la construcción de SQL compatible sigue quedando a cargo generalmente de los desarrolladores del proyecto.

Definición

Un proyecto puede tener asociada varias fuentes de datos, cada una representando a una base de datos distinta. Para administrar estas fuentes existe una sección en el Administrador llamada Datos, allí se puede detallar cómo está formada, los campos más importantes, además de los de conexión, son estos:

  • Identificador: es el nombre con el que esta fuente puede ser referenciada desde Toba.
  • Link instancia: Si se selecciona, los parámetros de conexión de este formulario no se tendrán en cuenta, en cambio los párametros de la instancia indicada abajo son los válidos. Esto evita la carga de los parámetros en estas definiciones delegando hacia un archivo php (ver definición de instancias).

Ejecución

Durante la ejecución las fuentes de datos contienen estos parámetros estáticos junto a una instancia de la librería ADODb para el acceso a la misma. Por performance las fuentes no se conectan hasta que sea necesario, esto es porque el objeto en ejecución la necesita o porque se utiliza en algún código ad-hoc. En este último caso, sabiendo que ningún otro procesamiento cargo esa fuente de datos, se debe abrir con la siguiente función global: 

abrir_fuente_datos($nombre_fuente);

Una vez disponible la fuente puede obtenerse el objeto ADODb asociado con la siguiente sentencia: 

toba::get_db($nombre_fuente0;

Donde $fuente es el nombre de la fuente de datos. Por ejemplo para poner en modo debug a esta librería es necesario ejecutar esta sentencia: 

toba::get_db($fuente)->debug = true;
