Instancias

Una instancia contiene los metadatos de un conjunto de proyectos, incluyendo los del mismo Administrador? (ya que TobaConsumeToba). Por esto recibe el nombre de instanciación de un proyecto, ya que un Proyecto puede estar contenido en muchas instancias diferentes.

Configuración

Una instancia generalmente se almacena en una base de datos propia. El lugar físico depende de las configuraciones del archivo toba/php/instancias.php. Para armar instancias propias y para la instalación inicial se puede tomar el archivo source:trunk/php/_instancias.php como base. El nombre de la instancia es la clave del arreglo asociativo, así si se quiere definir una nueva instancia nombre_instancia en la base nombre_base: 

<?php
    $instancia["nombre_instancia"][apex_db_motor] = "postgres7";
    $instancia["nombre_instancia"][apex_db_profile] = "localhost";
    $instancia["nombre_instancia"][apex_db_usuario] = "postgres";
    $instancia["nombre_instancia"][apex_db_clave] = "";
    $instancia["nombre_instancia"][apex_db_base] = "nombre_base";
?>

Composición

Una instancia puede estar formada por distintos proyectos, entre ellos siempre se encuentra el Administrador?. La instancia posee sectores comunes como los Usuarios que no pertenecen a un proyecto particular sino que son globales a la instancia y se relacionan con los distintos proyectos en diferentes roles. Una vez exportada la instancia la definición de estos usuarios se encuentra en el archivo datos_instancia.sql del proyecto toba. Este archivo no se encuentra versionado en Toba por lo que es responsabilidad de cada proyecto mantenerlo.

Creación de una instancia

La administración de las instancias se realiza bajo la consola. La consola está configurada con una instancia por defecto, la variable de entorno toba_instancia. Para ver la instancia actual ejecutar: 

echo $toba_instancia

Luego de definir las configuraciones de la instancia es posible cargar todos los proyectos en una instancia particular. Para esto se utiliza el siguiente comando: 

instancia iniciar

En caso de que la instancia se hubiera creado anteriormente, es necesario regenerarla: 

instancia regenerar

Exportación de la instancia

No es usual exportar la instancia completa, ya que entre los proyectos se encuentra el Administrador, mantenido por el grupo de desarrollo de Toba. Como resultado de esta exportación se va a sobreescribir el archivo datos.sql no sólo del proyecto propio sino también del mismo Toba. En cambio exportar la sección común de la instancia es recomendable para poder mantener en un archivo los datos de usuarios en esta instancia. Como Toba es un proyecto más, basta con exportar la información local del proyecto Toba: 

proyecto exportar_local toba
