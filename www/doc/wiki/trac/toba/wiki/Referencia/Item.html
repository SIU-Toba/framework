Item

Hablar de un ítem es referirse a la definición de una operación. La operación se encuentra incluída en un árbol de items dentro de una carpeta particular y su comportamiento está determinado por el patrón seleccionado y sus objetos contenidos.

Propiedades básicas

  • Identificador: El identificador permite referirse al ítem desde otro punto del administrador o desde el mismo código fuente. Se asigna automáticamente al crear el ítem.
  • Padre: Determina la carpeta en donde estará el ítem.
  • Tipo de Solicitud: Determina el entorno en donde será ejecutado el ítem. El entorno usual es uno Web (Solicitud de Browser) aunque es posible uno de consola, útil para ejecuciones automáticas o scripts administrativos.
  • Modelo de Página: Permite variar el HTML incluído antes y después del contenido del ítem, por ejemplo para agregar una barra de navegación o un pie de página particular. Ver Tipo de página.
  • Tipo de Comportamiento: Describe si la actividad del ítem está determinada por un script PHP en el sistema de archivos (o bien en la fuente de datos) o por un comportamiento predefinido de Toba. Si se trata de un script PHP, al seleccionar la opción se solicita el path del archivo (o su identificación en la fuente de datos).
  • Comportamiento: Siempre que el tipo de comportamiento sea Predefinido tendremos que seleccionar un comportamiento, que tendrá a su cargo la creación y ejecución de los objetos particulares asociados al ítem.
  • Menú y Orden: Si el ítem se marca como menú se podrá visualizar en la aplicación. Para esto es necesario que las carpetas que lo contienen estén en menú y el usuario logueado tenga permisos sobre estas carpetas y el mismo ítem.

Permisos

Para poder ejecutar un ítem es necesario asignar permisos básicos no sólo al ítem sino también a las carpetas en las que está contenido (ver detalles de permisos?).

Objetos asociados

Dependiendo del Patrón seleccionado, el ítem necesita distintos Objetos para su ejecución, desde el Administrador es necesario asignar estos objetos.

Creación y uso

Desde el mismo administrador es posible instanciar el ítem para hacer una edición en línea. También es posible simular la ejecución de los Objetos asociados al ítem en forma independiente.

Navegación entre ítems

La navegación entre ítems pasa por un módulo de Toba: el Vinculador?. El vinculador es el que determininará la posibilidad de acceso dependiendo de los permisos definidos en el ítem destino. Desde el administrador es necesario explicitar el destino de estos links y la forma que toman.
