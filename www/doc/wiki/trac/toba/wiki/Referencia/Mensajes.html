Mensajes y Notificaciones

Toba brinda mecanismos para hacer más uniforme la construcción de mensajes y notificarlos al cliente.

Construcción de Mensajes

Existe la posibilidad de centralizar el armado de mensajes, agregando plantillas de mensajes a nivel global (vista EXT) o a nivel de un objeto. El mensaje actúa como plantilla dejando marcas para su posterior instanciación. Estas marcas toman la forma de %posicion%: 

Esta es la %1% instancia de un mensaje global de Toba. Fecha de hoy: %2%.

Cuando el mensaje es global al proyecto, se lo busca invocando desde cualquier lado, utilizando el índice del mensaje como primer parámetro y los reemplazos de las marcas como segundo: 

<?php
    echo mensaje::get('indice', array('primera', date('d/M/Y'));
?>

Cuando el mensaje es definido desde el Administrador en el mismo objeto, sólo se pude invocar desde una extensión de este objeto de la siguiente forma: 

<?php
    echo $this->obtener_mensaje('indice', array('primera', date('d/M/Y'));
?>

El mensaje obtenido es el siguiente: 

Esta es la primera instancia de un mensaje global de Toba. Fecha de hoy: 13/Jun/2005

Notificación

Una vez construidos los mensajes, ya sea con la forma anterior o en forma manual, se los puede agrupa en una cola_mensajes para ser mostrados posteriormente al cliente. Esta agrupación a bajo nivel se hace con la clase cola_mensaje en php/nucleo/lib/cola_mensajes.php: 

<?php
    toba::get_cola_mensajes()->agregar("Error al procesar");
?>

Opcionalmente se puede incluir el tipo de mensaje como segundo parámetro ('error' o 'info'), por defecto es 'error'. Por ahora la única forma en que se muestra esta cola es con un alert de javascript detallando cada uno de los mensajes encolados.

Cuando el error se detecta desde la extensión un objeto toba la notificación se hace utilizando un servicio del mismo objeto: 

<?php
    $this->informar_msg("Error al procesar");
?>
