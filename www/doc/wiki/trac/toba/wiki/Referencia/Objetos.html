Objetos Toba

Una operación se define en base a los llamados objetos Toba. Estos objetos cubren distintos aspectos de una operación, categorizándolos según su función: interface, navegación, negocio o persistencia. Como todo elemento de Toba, el comportamiento particular de los objetos es determinado por:

  1. La definición estática objeto usando el administrador. Ver Definición de un objeto.
  2. La manipulación del objeto en ejecución a través de servicios. Ver Extensión de un objeto.

Elementos de interface (ei)

Los objetos de interface son componentes gráficos usuales en aplicaciones empresariales como formularios, filtros y cuadros. Dentro de Toba se los conoce como elementos de interface o su abreviación ei. Actualmente, los objetos de interface definidos son:

  • ei_formulario: Formulario de edición simple.
  • ei_formulario_ml: Formulario de edición de múltiples líneas.
  • ei_filtro: Filtro de selección.
  • ei_cuadro: Grilla para mostrar y seleccionar datos.
  • ei_calendario: Calendario para visualizar contenidos diarios y seleccionar días o semanas.
  • ei_arbol: Visualiza una estructura de árbol en HTML.
  • ei_archivos: Permite navegar el sistema de archivos del servidor bajo una carpeta dada.
  • ei_esquema: Muestra un esquema utilizando GraphViz.

Cada ei dispara un conjunto predefinido de Eventos, por ejemplo por defecto el formulario posee los eventos Alta, Modificación, Baja y Cancelar. Las capas inferiores toman estos eventos brindando comportamiento a cada ei, por ejemplo en el caso del formulario le daría una lógica de ABM.

Controladores de interface (ci)

Estos objetos son una abstracción de las distintas pantallas por las que atraviesa una operación. En cada uno de estos estados sirven como contenedores de otros objetos de interface y determinan la carga y comportamiento particular en torno a esta operación. Por determinar el flujo de la operación se los conoce como controladores de interface o su abreviación ci:

  • Controlador de Interface básico.

En los CI se escuchan los eventos generados por los EI u otros CI contenidos, almacenando en su memoria los datos necesarios para el siguiente pedido.

Controladores de negocio (cn)

Estos objetos sirven para desacoplar la logica de la interface de los servicios de la clases de negocio propias. Son utiles cuando existen muchas clases de negocio que necesitan llamarse desde la operacion o cuando existe una familia de operaciones que comparten sub-controladores de interface y necesita centralizarce la informacion cargada antes de procesarla. Esto necesita de mas explicacion.

Elementos que manejan datos (datos)

Estos objetos permiten modelar tablas y registros de una base relacional o algún medio que maneje datos tabulares, brindando servicios transaccionales a las capas superiores. En estos objetos se describen las estructuras y asociaciones de las tablas involucradas en una operación y la forma en que los registros serán sincronizados al final de la misma.

  • Introducción a los objetos de persistencia.
  • datos_tabla: Abstracción de una tabla
  • datos_relacion: Abstracción de una relación entre tablas.
  • Administrador de persistencia: Mecanismo de carga y sincronización con un medio de persistencia.
