Persistencia en Sesión

En la subclase específica de un objeto se pueden especificar aquellas propiedades que se desean mantener para el próximo pedido de página.

¿Porqué persistir propiedades en sesión?

Como se ve en el marco transaccional de una operación, se necesita una forma de recordar cosas en una intercambio cliente (browser) - servidor (php). En una programación ad-hoc se utilizan las llamadas variables de sesión de PHP, pero para organizar y optimizar el consumo de estas variables, cada objeto-toba se ofrece tomar la responsabilidad de administrar estas variables.

¿Cómo persistir estas variables?

Para esto se debe redefinir el método mantener_estado_sesion en la subclase del objeto-toba en cuestión. Por ejemplo el siguiente es un encabezado de una subclase de un CI: 

<?php
require_once('nucleo/browser/clases/objeto_ci.php');
class mi_ci extends objeto_ci
{
    protected $variable_1;
    protected $variable_2;
    
    function __construct($id)
    {
        //Zona apta para inicializaciones por defecto
        parent::__construct($id);     //Aquí se busca en la sesión las propiedades marcadas
        //Aquí ya se restauraron los valores de las propiedades marcadas
    }
    
    function mantener_estado_sesion()
    {
        $propiedades = parent::mantener_estado_sesion();
        $propiedades[] = "variable_1";
        $propiedades[] = "variable_2";
        return $propiedades;
    }
...
}
?>

Estas variables serán almacenadas cuando se termine la construcción de una página y restauradas en el siguiente pedido.

¿Que tipo de variables se pueden persistir?

Antiguamente sólo se podían persistir estructuras (tipos básicos, arreglos, etc.) y no objetos. Actualmente es posible persistir grafos completos de objetos con algunas consideraciones:

  • Si estos objetos poseen otros por composición, es posible que las clases de estos estén definidos en un archivo no incluido originalmente, para esto se debe utilizar la función __autoload de PHP que permite al sistema encontrar el archivo de esta clase antes de intentar reconstuir el objeto.
  • Si en esta composición existen referencias circulares es necesario definir __sleep en el objeto hijo, para anular el controlador y __wakeup en el padre para restablecerlo.
