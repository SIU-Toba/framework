Puntos de Acceso

¿Qué son?

Los Puntos de Acceso (abreviado PA) son configuraciones que definen el ingreso tanto al aplicativo de un proyecto como al administrador del mismo. Cada proyecto puede tener varios puntos de acceso, por defecto se proveen dos: una para la aplicación (aplicacion.php) y otro para el administrador (admin.php) y se ubican bajo el directorio www del proyecto.

¿Cómo crear uno?

Se recomienda tomar como modelo los puntos de acceso que están incluidos en el mismo Toba.

  • PA de la aplicación: source:trunk/www/aplicacion.php
  • PA del administrador: source:trunk/www/admin.php

Luego de modificar las configuraciones particulares es necesario notificar a Apache la existencia del nuevo PA. Para esto es necesario agregar al archivo directorio_apache/conf/toba.conf un alias al punto de acceso: 

    Alias /mi_proyecto "d:/toba/proyectos/mi_proyecto/www"
    <Directory "d:/toba/proyectos/mi_proyecto/www">
        SetEnv TOBA_DIR d:/toba
        Options Indexes MultiViews
            DirectoryIndex aplicacion.php
        AllowOverride None
        Order allow,deny
        Allow from all
    </Directory>

Notar que dentro del punto de acceso debemos setear el valor de la variable de entorno $TOBA_DIR, para que refiera al directorio donde se encuentra la instalación de Toba. Esto es necesario para el caso de varias instalaciones de Toba dentro de la misma máquina (ver casos de uso de deployment), porque cada una obviamente estará en un directorio diferente.

Configuraciones Importantes

  • Alias: La variable apex_pa_toba_alias debe contener el del alias correspondiente a la instalación de Toba a la cual pertenence la instancia. Esta directiva se puede configurar desde el mismo alias de apache incluyendo SetEnv TOBA_ALIAS alias}. Si el alias (en el archivo directorio_apache/conf/toba.conf) tiene asignado el nombre toba_prueba, se define así: 
    define("apex_pa_toba_alias", "toba_prueba");
  • Instancia: La variable apex_pa_instancia debe apuntar a la instancia que se quiere acceder, por ejemplo para acceder a una instancia particular toba_prueba: 
    define("apex_pa_instancia","toba_prueba");
  • Acceso: Para desarrollo es muy práctico evitar pedir un login en cada acceso, para eso se usa la variable apex_pa_validacion_debug. Importante: Desactivar esta configuración en deployment. 
define("apex_pa_validacion_debug",1)
  • Encriptación: Para maximizar la seguridad es posible encriptar el querystring seteando la variable apex_pa_encriptar_qs: 
    define("apex_pa_encriptar_qs",1);
  • Log: Es posible elegir el medio y el nivel de logging interno de Toba. Esto es útil para hacer trazas de desarrollo. 
    # Nivel de log a ARCHIVO
define("apex_pa_log_archivo",1);
define("apex_pa_log_archivo_nivel",2);
# Nivel de log a la DB
define("apex_pa_log_db",1);
define("apex_pa_log_db_nivel",2);
# Nivel de log a la pantalla
define("apex_pa_log_pantalla",1);
define("apex_pa_log_pantalla_nivel",7);

Ver TobaSeguridad? para más información acerca de los mecanismos de seguridad existentes.
