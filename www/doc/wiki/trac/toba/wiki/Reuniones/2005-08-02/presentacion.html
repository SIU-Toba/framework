Presentacion del SIU-Toba

Caracteristicas

  • Tiempo: 20 min.
  • Destinatario: Humano con conocimientos de sistemas que no es necesariamente programador.

Secciones

  1. (5 min.): Explicar los principios de la herramienta de una forma simple.
  2. (5 min.): Mostrar el tipo de interaces que se pueden lograr con ejemplos.
  3. (5 min.): Comentar algunos detalles interesantes.
  4. (5 min.): Hablar sobre el esquema de trabajo.

Contenidos

1. Principios basicos

  • Introducción [Apoyada sobre un PPT o algo asi]
    • El toba es una herramienta (o ambiente) para hacer aplicaciones compuestas por operaciones tratando de resolver la mayor cantidad de aspectos en forma declarativa.
    • La teconología utilizada se basa en librerías y herramientas abiertas con una amplia comunidad de usuarios
    • Las operaciones se agrupan en módulos y se publican en un menu, asignandose estilos visuales particulares y permisos a nivel de grupo de usuarios.
    • Ya dentro de una operación se utilizan una serie de elementos con responsabilidades distribuidas en capas, con la siguiente división:
      • Elementos de interface (formularios, cuadros, etc).
      • Controladores del flujo de pantallas (lugar para atacar la complejidad del comportamiento).
      • Elementos relacionados con la persistencia en BD relacionales.
    • El armado de una operación comienza agrupando distintos componentes visuales en pantallas con un comportamiento definido en un controlador. Estas pantallas se secuencian y anidan brindando marcos transaccionales que culminan con la sincronización directa con la base de datos o con objetos de negocio propios de la aplicación.
    • Entrar en un nivel mas de detalle, [En las pantallas se acompaña con el]
    • La construcción declarativa impone un balance entre metadatos y código:
      • Utilizar la parte PHP de los editores
      • Explicar la relacion PHP/JS.
  • Alcance de la herramienta?
    • Permite enfocar el desarrollo en la logica de negocio y no en la infraestructura. No se necesita un grupo especializado en infraestructura permitiendo construir un grupo más orientado a concretar las necesidades de negocio.
    • Apunta a lograr un balance adecuado entre tiempo de desarrollo y calidad del producto.

2. Ejemplos de interfaces

  • Formas de navegacion
  • Colapsado, resumen y ocultamiento
  • Formularios ML
  • Validaciones en línea

3. Temas, cosas interesantes

  • toba consume toba (ideologia)
  • Ventana transaccional web
  • Editor de CSS
  • Debug Javascript
  • Seguridad

4. Esquema de trabajo

  • El equipo de desarrollo esta distribuido:
    • Unificado por el control de versiones
    • Politica orientada a la integracion continua, el deploy es continuo.
    • testing automatico
    • trac
  • Pasado: Impacto de la historia en el desarrollo.
    • Como fue creciendo el sistema incorporando ideas de todos.
    • El desarrollo fue guiado por la utilizacion de proyectos concretos.
  • Futuro: Crear un lugar donde un grupo de desarrolladores se reune a resolver un problema de dominio especifico. Para alcanzar este objetivo el SIU-Toba deberia brindar una serie de servicios que faciliten el trabajo cooperativo de un grupo distribuido.
    • Organizacion del desarrollo.
    • Esquemas de intercambio.
    • Versiones del sistema especificas por desarrollador (de una aplicacion final o del toba mismo).
    • Crear catalogos de formas estandar de atacar problemas comunes.
